# 테스트 카페 스케줄 큐 추가

qwzx16 서비스 계정의 테스트 카페에 스케줄링합니다.
원고 스타일은 **애니** 기반입니다.

## 작업 흐름

### 1단계: 입력 확인

사용자에게 아래 정보를 확인합니다:

- **대상 카페**: 벤타쿠 / 으스스 / 다향만리 / 복수 선택 가능
- **카페별 글 수**: 예) 벤타쿠 3개
- **마감 시간**: 기본 오후 10시 (22:00)

### 2단계: 키워드 선정

1. **일상 키워드 생성**: 카페 성격에 맞는 자연스러운 키워드 직접 생성
   - **벤타쿠**: 밴드/음악/공연/굿즈/앨범 관련
   - **으스스**: 자유 주제
   - **다향만리**: 차/티타임/찻잔 관련
2. **필수**: 구체적 고유명사(밴드명, 앨범명, 차 브랜드 등) 포함
3. 웹 검색으로 최근 화제인 밴드/앨범/공연 정보 확인 후 반영

### 3단계: 시간 배정

- 현재~마감까지 균등 분배
- **계정 간격 규칙**: 같은 계정은 최소 40분 이상 간격 유지
- 3개 계정이 writer + commenter 겸용이므로 부하 고려

### 4단계: 사용자 확인

아래 형식의 표로 전체 스케줄을 보여주고 확인받습니다:

```
## 카페명

| 키워드 | 카테고리 | 계정 | 시간 |
|--------|---------|------|------|
```

수정 요청이 있으면 반영 후 재확인.

### 5단계: 큐 추가 실행

확인 받으면 `scripts/run-schedule.ts`의 `SCHEDULE` 배열과 `LOGIN_ID`를 업데이트하고 실행합니다:

**중요**: `LOGIN_ID`를 `"qwzx16"`으로 변경해야 합니다.

```typescript
const LOGIN_ID = process.env.LOGIN_ID || "qwzx16";
```

```bash
npx tsx --env-file=.env.local scripts/run-schedule.ts
```
